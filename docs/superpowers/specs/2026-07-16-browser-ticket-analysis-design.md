# 浏览器工单动态分析设计

## 目标

将当前固定样本静态报告升级为浏览器内动态分析工具。用户可加载内置示例或选择本地 JSON 文件，页面按实际数据重算指标、图表、异常与结论，且不会把数据上传到服务器。

## 文件与职责

- `doc/data/example-tickets.json`：可由 GitHub Pages 访问的内置示例，内容与 `source/task5_tickets.json` 一致。
- `doc/app.js`：数据校验、聚合分析、异常判定和 DOM 渲染。纯分析函数同时导出给 Node 测试，浏览器端挂载初始化函数。
- `doc/index.html`：加载控件、状态消息、动态报告容器和本地脚本引用；不再硬编码报告数值。
- `tests/analyze-tickets.js`：验证数据校验、聚合结果、异常规则与边界情况。
- `tests/verify-report.js`：保留页面离线资源与 Pages 配置检查，调整为验证动态入口和本地脚本。

## 数据入口

1. 页面初始化时通过 `fetch('./data/example-tickets.json')` 自动加载内置示例。
2. “加载内置示例”按钮重新加载该文件。
3. “选择本地 JSON”触发隐藏的 `<input type="file" accept="application/json,.json">`；文件以 `FileReader.readAsText` 读取并解析。
4. 每次加载成功后，状态区说明来源（内置示例或本地文件）、有效工单数量和时间范围；本地数据只在当前浏览器内存中处理。

## 校验规则

文件必须是非空 JSON 数组。每条工单要求：非空字符串 `ticket_id`、可解析的 `created_at`、非空字符串 `category`、`priority` 为高/中/低、非负有限数值 `resolution_time_hours`、1 至 5 的有限数值 `satisfaction`、非空字符串 `channel`、布尔值 `is_resolved`。`description` 可以为空或缺失，不影响聚合分析。

解析失败、非数组、空数组或任一条记录不合格时，在加载区显示具体行号和字段错误，保留上一次成功渲染的报告，不更新当前数据。

## 分析与渲染

`analyzeTickets(tickets)` 返回以下数据：

- 总量、时间范围、未解决率、平均处理时长、平均满意度、高优先级数量。
- 按日期趋势、分类、优先级、解决状态、渠道的数量和占比。
- 分类与渠道的未解决数、平均处理时长、平均满意度和高优先级数。
- 异常信号和建议。

报告一次性重新渲染为现有的指标卡、每日趋势 SVG、渠道面板、四张环形图、分类服务质量表、异常卡片和分析维度说明。分布项目按数量降序、名称次级排序，所有图例均显示项目、数量和占比，SVG 具备完整 `aria-label`。

## 动态异常规则

- 问题结构：数量最多的分类显示数量、占比、高优先级占比，并建议排查该业务链路。
- 未解决风险：有未解决工单的分类中，未解决率最高者显示数量、未解决率、平均处理时长和建议。
- 渠道体验：平均满意度最低的渠道显示数量、处理时长和满意度；仅比较样本中实际出现的渠道。
- 时间趋势：当至少有 6 个日期时，比较最近 3 天与此前 3 天的日均量；近期日均高出 20% 及以上时标记增长，低于 20% 及以上时标记下降，其余标记稳定。少于 6 个日期时提示样本不足以判断短期趋势。

这些规则是样本内风险排序，不构成增长预测或长期因果判断。页面始终展示数据范围，并在样本不足时说明限制。

## 验证

- 以给定 50 条样本验证现有统计结果与四类分布保持一致。
- 验证有效自定义样本会产生相应统计与异常；无效 JSON、空数组、字段类型错误被拒绝，且不会覆盖已成功分析的结果。
- 验证页面提供两个数据入口、加载状态、`app.js` 和内置样例文件，且没有外部资源。
- 用本地 HTTP 服务验证内置示例可加载；文件选择流程由浏览器端 `FileReader` 实现。
