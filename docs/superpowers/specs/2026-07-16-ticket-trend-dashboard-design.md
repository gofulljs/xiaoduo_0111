# 客服工单趋势分析静态站点设计

## 目标

基于 `source/task5_tickets.json` 的 50 条工单，在根目录 `doc/` 交付可直接部署到 GitHub Pages 的静态分析站点。站点帮助客服主管在单页内识别趋势、服务风险和优先行动。

## 交付物

- `doc/index.html`：单页响应式仪表盘，内置样式、数据和图表，不依赖网络或构建工具。
- `doc/README.md`：分析维度、关键趋势和异常、AI 工具使用说明。
- `.github/workflows/deploy-pages.yml`：将 `doc/` 发布到 GitHub Pages 的工作流。

## 信息架构

1. 页首说明数据范围和统计口径。
2. 摘要指标显示总量、未解决率、平均处理时长和平均满意度。
3. 趋势区展示每日工单量及主要分类变化。
4. 分布区展示分类、优先级、渠道及服务质量的交叉分析。
5. 异常区按风险等级展示信号、判断依据、关联工单和建议动作。
6. 方法区解释分析维度的决策价值、异常阈值及数据限制。

## 分析维度与规则

| 维度 | 指标 | 决策价值 |
| --- | --- | --- |
| 时间趋势 | 每日总量、分类日占比 | 发现突然增长和持续高发问题，便于排班及专项排查。 |
| 问题结构 | 分类数量、占比、优先级构成 | 判断问题规模与紧急程度，分配产品、支付、物流等处理资源。 |
| 服务质量 | 未解决率、平均处理时长、平均满意度 | 识别积压、处理瓶颈和客户体验风险。 |
| 关联关系 | 分类 × 状态、优先级、渠道、满意度 | 判断风险是否集中在某类问题或服务链路。 |

异常信号必须同时列出实际数值和依据。采用以下固定规则：

- 高频且持续：某分类在最近连续 3 天出现，且累计量位列分类第一。
- 未解决风险：某分类未解决率高于全量未解决率，且存在未解决高优先级工单。
- 服务体验风险：平均处理时长高于全量平均值且平均满意度低于全量平均值。
- 严重事件聚集：同一分类的高优先级工单占该分类半数以上，或描述中反复出现相同业务症状。

## 实现边界

- 使用原生 HTML、CSS 和 JavaScript；图表使用 SVG 或 CSS，不使用 CDN。
- 分析结果基于题目给定的静态样本，页面不提供上传、筛选或实时更新能力。
- 页面会包含移动端单列布局和无障碍的文字替代说明。
- GitHub Actions 使用官方 Pages actions；仓库需在 Settings > Pages 中将 Source 设置为 GitHub Actions。

## 验证

1. 以脚本独立重算页面中的每个统计值，并与 JSON 数据比对。
2. 检查 `index.html` 为独立可打开的有效 HTML，且不包含外部资源链接。
3. 检查工作流语法、发布路径和权限配置。
4. 在桌面和窄视口检查图表、表格与异常卡片的可读性。
