# 客服工单维度分布图设计

## 目标

在现有静态客服工单报告中增加四张环形图，使主管能快速查看问题结构、紧急程度、积压情况和渠道分布，并直接获得基于样本数据的结论。

## 范围

仅修改 `doc/index.html` 和 `tests/verify-report.js`。不引入第三方库、网络资源、筛选器或交互状态；所有图形均为内联 SVG，页面继续可离线打开。

## 布局

在每日工单趋势图之后、服务质量表之前增加“维度分布与结论”区域。桌面端使用两列四卡布局，最大宽度 720px 时切换为单列。每张卡片包含标题、SVG 环形图、颜色对应的图例（分类、数量、占比）和一条结论。

## 图表与结论

| 维度 | 分布 | 结论 |
| --- | --- | --- |
| 问题分类 | 支付问题 16（32%）、退款退货 13（26%）、物流查询 8（16%）、商品咨询 5（10%）、账号问题 4（8%）、投诉 4（8%） | 支付与退款退货合计 29 条、占 58%，应作为支付链路和退款时效治理的优先方向。 |
| 优先级 | 高 31（62%）、中 13（26%）、低 6（12%） | 高优先级占 62%，主管应先保障风险工单的响应能力。 |
| 解决状态 | 已解决 42（84%）、未解决 8（16%） | 5 条未解决工单属于退款退货，占未解决工单 62.5%，应优先清理退款积压。 |
| 来源渠道 | 在线 34（68%）、电话 16（32%） | 电话渠道仅占 32%，但平均处理时长 31.63 小时、平均满意度 1.94，均弱于总体水平。 |

## 可访问性与数据口径

- 每个 SVG 使用 `role="img"` 和描述完整分布的 `aria-label`。
- 图例不只依赖颜色，必须写明维度项、数量与百分比。
- 百分比以该维度总样本 50 条为分母，显示为整数；未解决退款占比以 8 条未解决工单为分母，显示为 62.5%。
- 结论只陈述源数据直接支持的比较和建议，不推断未提供字段的原因。

## 验证

`tests/verify-report.js` 从源 JSON 重算四个维度的数量和百分比，断言四个环形图的语义标识、SVG、图例、结论和无障碍描述均存在且与重算结果一致。保留现有报告、README 和 Pages 工作流校验。
